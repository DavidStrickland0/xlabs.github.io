Measure the view and its content to determine the measured width and the measured height. This method is invoked by M:Android.Views.View.Measure(System.Int32, System.Int32) and should be overriden by subclasses to provide accurate and efficient measurement of their contents.

CONTRACT: When overriding this method, you must call M:Android.Views.View.SetMeasuredDimension(System.Int32, System.Int32) to store the measured width and height of this view. Failure to do so will trigger an IllegalStateException, thrown by M:Android.Views.View.Measure(System.Int32, System.Int32). Calling the superclass' M:Android.Views.View.OnMeasure(System.Int32, System.Int32) is a valid use.

The base class implementation of measure defaults to the background size, unless a larger size is allowed by the MeasureSpec. Subclasses should override M:Android.Views.View.OnMeasure(System.Int32, System.Int32) to provide better measurements of their content.

If this method is overridden, it is the subclass's responsibility to make sure the measured height and width are at least the view's minimum height and width (M:Android.Views.View.get_SuggestedMinimumHeight and M:Android.Views.View.get_SuggestedMinimumWidth).
